    The SLA Matrix shows how much of each batch's contracted time window actually got used. Every workflow gets a buffer % — the gap between actual runtime and the SLA ceiling. That number is what tells you how close you are to missing a delivery commitment.
    
            
    Below 15% buffer means one slow night tips you into breach territory. LONG_JOB (15–40%) is worth watching. AT_RISK is a red flag. BREACH means the contracted window was already missed — that requires a fix or a signed waiver before PE sign-off.
    
            
    Healthy looks like: all workflows above 40% buffer, runtime flat across days, no workflow repeating BREACH. If the same workflow shows BREACH or AT_RISK twice in a row, it's a systemic problem, not a one-off.
